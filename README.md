Building AI systems for regulated industries.

I work on LLM applications where getting things wrong has real consequences — financial compliance, regulatory interpretation, investment decisions. Most AI demos ignore these domains because they're hard. That's why they're interesting.

**Currently building:**
- [RegAI](https://github.com/balabommablock-cpu/regai) — RAG pipeline for financial regulations. Domain-specific chunking that preserves table context and amendment chains.
- [Boredfolio](https://github.com/balabommablock-cpu/boredfolio) — Mutual fund intelligence for India. Plain English → fund analysis with citations.
- [LLM-Eval-BFSI](https://github.com/balabommablock-cpu/llm-eval-bfsi) — Benchmarking LLMs on regulated financial tasks.

**Contributing to:**
- [anthropics/financial-services-plugins](https://github.com/anthropics/financial-services-plugins/issues/15) — Indian BFSI plugin for mutual fund, regulatory, and tax workflows
- [anthropics/claude-cookbooks](https://github.com/anthropics/claude-cookbooks/issues/407) — Domain-specific RAG cookbook for financial regulations
